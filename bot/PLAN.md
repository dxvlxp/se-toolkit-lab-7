The goal of this task is to create a clean bot scaffold before implementing real features. The architecture should separate transport logic from business logic, so command handlers can be tested without Telegram. The bot entry point will support two modes: Telegram mode for real users and `--test` mode for offline verification in the terminal. The handler layer will contain pure functions that accept text input and return text output. This allows the same logic to be reused by both Telegram updates and CLI tests.

The `services` layer will be responsible for external integrations. In later tasks, it will contain an LMS API client for backend requests and an LLM client for intent routing. For now, these services can stay as placeholders, but the structure should already exist so future features can be added without rewriting the project.

Configuration will be loaded from `.env.bot.secret`, with `.env.bot.example` serving as the template. The deployment plan is to keep the bot running on the VM using `uv run bot.py`, verify commands through `--test`, and then validate that `/start` works in Telegram. This scaffold makes Task 2 and Task 3 easier because the command routing and program entry points are already in place.
